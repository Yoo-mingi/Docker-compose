Compose - 단일 서버에서 여러 컨테이너를 프로젝트 단위로 묶어서 관리하기 위한 Tool.
docker-compose.yaml 이라는 YAML파일을 통해 "명시적"으로 관리

장점 :
  프로젝트 단위로 도커 네트워크와 볼륨 관리
  프로젝트 내 서비스 간 의존성 정의 가능
  프로젝트 내 서비스 디스커버리 자동화
  손 쉬운 컨테니어 수평확장

3가지 핵심 key : 

프로젝트 : 
도커 컴포즈에서 다루는 워크스페이스 단위. 함꼐 관리하는 서비스 컨테이너의 묶음. 프로젝트 단위로 기본 도커 네트워크가 생성

서비스 :
도커 컴포즈에서 컨테이너를 관리하기 위한 단위. scale를 통해 서비스 컨테이너의 수 확장 가능

컨테이너 :
서비스를 통해 컨테이너 관리


프론트그라운드로 프로젝트 실행 : 
docker-dompose up

백그라운드로 프로젝트 실행 : 
docker-compose up -d

프로젝트 이름 "AAAA" 로 프로젝트 실행 : 
docker-compose -p AAAA up -d

프로젝트 내 컨테이너 종료 및 제거
docker-compose down

프로젝트 내 컨테이너, 네트워크, 볼륨 종료 및 제거
docker-compose down -v
